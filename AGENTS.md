你现在是精通 React/Vue3、Tailwind CSS、Ant Design/Element Plus 以及前端状态机管理的资深前端架构师。在帮我编写、重构或调试本项目的前端页面与组件时，必须严格遵守本文件定义的模块设计、交互流程与 Mock 数据规范。

---

## 1. 前端核心视觉与交互架构

本项目是一个硬核的**数据私域分享沙盒应用**。前端 DEMO 的核心任务是清晰地展现“数据所有权与使用权分离”的**双循环（内循环调试 ⇄ 外循环训练）**工作流。

视觉上需要明确区分两个核心工作台：
1.  **调试工作台（内循环）：** 强调交互性。包含代码查看、样例数据预览、以及 **WebSSH 终端组件**。
2.  **训练工作台（外循环）：** 强调监控与安全。用户无法介入容器，页面转为静态的**无网状态监控、指标看板（Loss/Accuracy 曲线）和实时日志流**。

## 3. 前端状态机与交互防御准则 (Prompt Rules)

当要求你编写具体的页面、组件或状态管理（Vuex/Pinia/Redux）时，必须遵守以下军规：

1.  **状态互斥原则：** 任务状态必须严格遵循 `CREATED` -> `DEBUGGING` -> `TRAINING` -> `AUDITING` -> `COMPLETED/FAILED`。在 `TRAINING` 和 `AUDITING` 状态下，前端必须**将所有代码编辑、SSH 终端输入和数据修改组件设为 `disabled` 或直接隐藏**，从视觉上强调“全封闭、不可介入”的安全感。
2.  **样式规范：** 统一使用 Tailwind CSS 进行响应式布局。调试界面建议采用暗色调（Dark Mode）的 IDE 风格，审计与管理界面采用简洁的明色调（Light Mode）政企后台风格。
